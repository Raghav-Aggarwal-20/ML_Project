How it works

  1. EDA & data validation to understand distributions, detect issues, and confirm feature usefulness.
  2. Train/val pipeline with consistent preprocessing (categorical encoding + numeric scaling) used for both training and inference.
  3. Model training + hyperparameter tuning to select the best performing model using evaluation metrics.
  4. Persist artifacts (trained model + preprocessing pipeline) for reproducible, production-style inference.
  5. Deploy with CI/CD: Dockerize the app, push to AWS ECR, and run on EC2 via a self-hosted GitHub Actions runner.
